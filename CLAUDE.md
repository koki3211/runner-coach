# Runner Coach — 開発指針

## プロダクトの核となる4つの価値

1. **AIが根拠を持ってメニューを提案してくれる安心感** — 目標タイムからペースを自動算出し、フェーズ分け・漸進性過負荷・回復週を組み込んだ科学的なプランを生成する
2. **淡々とこなすためのシンプルなUX** — 今日やることが一目でわかり、余計な操作なく完了できる。機能追加よりも動線のシンプルさを優先する
3. **リアルな仲間がそれぞれ頑張っている姿から刺激をもらう** — 友達のストリークや今日の進捗が自然に目に入り、無言のモチベーションになる
4. **練習メニューをこなしたときの自己肯定感の上昇** — 完了時の演出、ストリーク、進捗リングで「やった」という実感を積み上げる

すべての設計判断・機能追加・UIの変更はこの4つに照らして評価すること。これらを損なう変更は改悪とみなす。

## デプロイ時の必須ルール

- **sw.js のキャッシュバージョンを必ずインクリメントする** — `index.html`・`app.js`・`social.js` など ASSETS に含まれるファイルを変更したら、`sw.js` の `CACHE` 定数のバージョン番号を +1 すること。これを怠るとユーザーに古いキャッシュが配信され続ける。
